## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / disclaimer / label / input prefix | 14–16px (`text-sm` / `text-base`) | 400 (regular) | 1.5 |
| Section heading / result value / metric label | 36–48px (`text-4xl md:text-5xl`) / 36px (`text-4xl`) / 16px (`text-base`) | 900 (black) | 1.2 heading / 1.1 result / 1.5 metric |

Declared weights: **400 (regular)** and **900 (black)** only.

Role mapping:
- Weight 400: body text, disclaimer, input label, input prefix ("R$"), accumulated estimate text
- Weight 900: section heading (`font-black tracking-tight`), result display values, metric card labels

Notes:
- Section heading uses `font-black tracking-tight` — consistent with all other landing headings.
- Result values are display-size numbers in `text-destructive` color — same weight as stat counters in the Stats Section.
- Disclaimer text uses `text-sm text-muted-foreground` — matches the trust signal pattern.

**Source:** Observed from `src/app/page.tsx` h2 pattern and Stats Section number size.

## Interaction Contract

### Input Behavior
- Input type: `text` (not `number`) to allow manual thousand-separator formatting
- Prefix: `R$` displayed as non-interactive label to the left of the input field, inside or adjacent to the input
- Placeholder: `"Ex: 50.000"`
- Formatting: Strip non-numeric characters on input change; re-format with `.` thousand separator for display (e.g. `50000` → `50.000`)
- Calculation trigger: `onChange` with debounce of 300ms
- No "Calculate" button — results update in real time

### State Machine
| State | Trigger | Visual |
|-------|---------|--------|
| Empty | Initial / input cleared | Result cards hidden; CTA hidden; only input and heading visible |
| Calculating | User typing (within debounce window) | No loading indicator — results update silently |
| Results | Valid numeric value >= 1 entered | Both result cards appear; accumulated estimate text appears; CTA appears |
| Invalid | User types 0 or clears field | Results hidden; CTA hidden |

### Result Cards Layout
- Two cards side by side on md+ screens, stacked on mobile
- Card 1 — "Perda por hora de downtime": icon `Clock` (or `Timer`), value = `receita / 30 / 24`, formatted as `R$ X`
- Card 2 — "Impacto de queda de 20% nas vendas": icon `TrendingDown`, value = `receita * 0.20`, formatted as `R$ X`
- Below cards: accumulated estimate text — `"Com 5h/mês de downtime: R$ X"` — small muted font
- Below accumulated estimate: disclaimer — `"Estimativa baseada em médias do setor"` — `text-xs text-muted-foreground`

### CTA Appearance
- Appears only after user enters a valid value (controlled by state)
- Label: `"Começar Grátis — Proteja Suas Vendas"`
- Links to `/auth/signup`
- Style: same gradient button pattern as Hero CTA (`bg-gradient-to-r from-primary to-purple-600`)

### Icon Assignments
| Icon | Lucide name | Usage |
|------|-------------|-------|
| TrendingDown | `TrendingDown` | Result card — perda por hora |
| AlertTriangle | `AlertTriangle` | Result card — queda de 20% |
| Clock | `Clock` | Already imported in page.tsx — section badge or input label |

## Transition / Animation Contract

- Result cards and CTA: no entrance animation — they appear/disappear with conditional rendering
- Input formatting: synchronous on each keystroke (after debounce, result values update)
- Metric numbers: plain text replacement — no count-up animation (keep it simple per CONTEXT.md scope)
- Cards on hover: `hover:shadow-2xl hover:border-destructive/50 transition-all duration-300` — consistent with other landing cards but using destructive border on hover instead of primary

**Source:** Claude's Discretion area (animation). "Simples" preference from CONTEXT.md.
