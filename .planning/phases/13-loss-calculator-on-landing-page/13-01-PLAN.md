# Phase 13 Plan 01: Loss Calculator Widget on Landing Page

## Objective

Add an interactive downtime loss calculator to the landing page. Visitors enter their monthly revenue and instantly see estimated hourly downtime losses and the impact of a 20% sales drop.

## Context

- Landing page: `src/app/page.tsx` (server component, 1196 lines)
- Calculator must be a Client Component (uses useState for interactivity)
- Design system: TailwindCSS + Radix UI + shadcn/ui (Card, Input, Badge, Button)
- Visual language: gradients from-primary to-purple-600, border, bg-muted/50, rounded-2xl
- Existing components in `src/components/` for reference

## Tasks

### Task 1: Create LossCalculator component

Create `src/components/loss-calculator.tsx` as a Client Component.

**Behavior:**
- Input field: monthly revenue in BRL (R$)
- Calculates hourly loss = monthlyRevenue / 720 (30 days * 24h)
- Calculates 20% sales drop impact = monthlyRevenue * 0.20
- Shows both results as formatted currency (Intl.NumberFormat BRL)
- Results update live as user types
- Shows "R$ 0,00" when input is empty/zero

**Visual design:**
- Card with gradient border effect, consistent with existing feature cards
- Red/destructive color theme for loss figures (emphasis on urgency)
- Input labeled "Seu faturamento mensal (R$)" with placeholder "ex: 50.000"
- Two result cards side by side: "Perda por hora de downtime" and "Impacto de queda de 20%"
- CTA button at bottom linking to /auth/signup

**Done criteria:**
- Component file created with `'use client'` directive
- Input updates trigger live recalculation
- Both loss figures display correctly formatted
- TypeScript compiles without errors

### Task 2: Wire LossCalculator into landing page

Insert the `<LossCalculator />` section into `src/app/page.tsx` between the Stats section (line ~205) and the Features section (line ~207).

**Done criteria:**
- `LossCalculator` imported into page.tsx
- Section added with heading "Quanto Você Perde com Downtime?"
- Section uses `id="calculator"` for anchor linking
- Page renders correctly (no TypeScript errors)

## Verification

- `src/components/loss-calculator.tsx` exists with `'use client'`
- `src/app/page.tsx` imports and renders `<LossCalculator />`
- Calculator section has `id="calculator"`

## Success Criteria

1. A visitor can type their monthly revenue and immediately see estimated hourly downtime loss
2. Calculator also shows the 20% sales drop impact
3. Widget uses same visual language as the rest of the landing page
