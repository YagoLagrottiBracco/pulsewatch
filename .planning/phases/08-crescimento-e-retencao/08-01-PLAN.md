# Plan: Phase 8 — Crescimento e Retenção

## Objective
Aumentar conversão free→pago e reduzir churn com onboarding gamificado, programa de indicações, reengajamento de inativos, upsell contextual e NPS automático.

## Tasks

### 1. Migration do banco de dados
- Tabela `onboarding_steps` com RLS
- Tabela `referrals` com tracking de código, status e recompensa
- Tabela `nps_surveys` (score 0-10, feedback, trigger_type)
- Tabela `upsell_events` (shown/clicked/dismissed por feature)
- Colunas extras em `user_profiles`: `referral_code`, `referred_by`, `trial_bonus_days`, `last_active_at`, `reengagement_sent_at`, `nps_sent_at`, `nps_completed`

### 2. Serviços backend
- `services/nps.ts` — cálculo NPS, geração de email, envio em batch
- `services/onboarding.ts` — progresso de steps, pontos, bonus de trial
- `services/reengagement.ts` — detecção de inativos, email de reativação
- `services/referral.ts` — geração de código, estatísticas, processamento de conversão

### 3. API Routes
- `POST/GET /api/nps` — submit resposta + buscar histórico
- `GET /api/nps/respond` — resposta via link de email
- `GET /api/onboarding` — progresso do usuário atual
- `POST /api/referral` — criar convite, buscar stats
- `POST /api/upsell` — registrar evento (shown/clicked/dismissed)
- `GET /api/cron/nps` — cron job: enviar NPS aos 30 dias de assinatura
- `GET /api/cron/reengagement` — cron job: reengajar free inativos há 7 dias

### 4. Componentes UI
- `OnboardingWidget` — card no dashboard com progress bar, steps e badge de bonus
- `NpsModal` — dialog com scale 0-10 e textarea de feedback
- `UpsellModal` — dialog contextual com benefícios do plano necessário

### 5. Integração no Dashboard
- Importar `OnboardingWidget` e `NpsModal` no `dashboard/page.tsx`
- Detectar `?nps_score=N` na URL (vindo do email) e abrir modal pré-preenchido

### 6. Página de indicação
- `app/referral/page.tsx` — página com código do usuário, stats e form de convite
