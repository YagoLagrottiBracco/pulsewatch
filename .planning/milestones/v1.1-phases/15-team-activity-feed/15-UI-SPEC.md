## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside feed entry (`gap-1`), Clock icon to timestamp text |
| sm | 8px | Gap between icon bubble and text block in each feed entry (`gap-2`), between action label and badge |
| md | 16px | Default element spacing within card bodies, tab content padding |
| lg | 24px | Section spacing between page header and tab content (`space-y-6`) |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Vertical padding in empty state (`py-12`) |
| 3xl | 64px | Page-level vertical rhythm (not used in this phase) |

Exceptions: Action icon bubble uses `p-2` (8px padding) matching the established `activity/page.tsx` pattern (`<div className="p-2 rounded-lg bg-muted">`). Feed entries have `pb-4 border-b last:border-0` matching existing activity log entry pattern.

**Source:** activity/page.tsx — observed `space-y-6`, `space-y-4`, `gap-4`, `gap-2`, `py-12`, `p-2` patterns. Carry-forward from Phase 14 spacing contract.

## Interaction Contracts

### activity/page.tsx — Tab Structure

The page retains its current heading "Registro de Atividades" and subtitle. Below the heading, two tabs appear:

```
<Tabs defaultValue="my-activity">
  <TabsList>
    <TabsTrigger value="my-activity">Minha Atividade</TabsTrigger>
    {isOwner && <TabsTrigger value="team">Time</TabsTrigger>}
  </TabsList>
  <TabsContent value="my-activity">
    {/* existing log list — unchanged */}
  </TabsContent>
  {isOwner && (
    <TabsContent value="team">
      {/* new team feed */}
    </TabsContent>
  )}
</Tabs>
```

Tab visibility rule: The "Time" tab is rendered only when `isOwner === true`. Non-owner users see only "Minha Atividade" with no TabsList rendered (or TabsList with a single entry — prefer no Tabs wrapper at all for non-owners to avoid unnecessary chrome).

Default active tab: `defaultValue="my-activity"`. The owner lands on their own activity first and can navigate to "Time" explicitly.

### Team Feed — Card Layout

The team feed renders inside `TabsContent value="team"` as a single Card, mirroring the existing "Ultimas Atividades" card structure.

**Card header:**
- `CardTitle`: "Atividade do Time"
- `CardDescription`: "Ultimas 50 acoes dos membros" (no accent, plain Portuguese)

**Feed entry layout** (matches existing entry pattern in activity/page.tsx):

```
<div className="flex items-start gap-4 pb-4 border-b last:border-0">
  <div className="p-2 rounded-lg bg-muted">
    {getTeamActionIcon(log.action)}   {/* Eye / Trash2 / Archive */}
  </div>
  <div className="flex-1 space-y-1">
    <div className="flex items-center gap-2">
      <p className="font-medium">{getActionLabel(log.action)}</p>
      <Badge variant="outline" className="text-xs">Alerta</Badge>
    </div>
    <p className="text-sm text-muted-foreground">
      {memberDisplayName}             {/* email or "Voce" */}
    </p>
    <div className="flex items-center gap-2 text-xs text-muted-foreground">
      <Clock className="h-3 w-3" />
      {relativeTimestamp}             {/* e.g., "ha 2h" */}
    </div>
  </div>
</div>
```

**Member display name rule:** If `log.user_id === currentUser.id`, render "Voce". Otherwise render the `email` field from `account_members` joined on the log's `user_id`.

**Timestamp format:** Relative ("ha 2h", "ha 3 dias"). Use a simple inline helper: if < 60 min → "ha Xmin", if < 24h → "ha Xh", if < 7 days → "ha X dias", else → absolute `toLocaleDateString('pt-BR')`. This matches the informal, conversational tone of the dashboard.

**Query limit:** 50 entries (hardcoded in Supabase `.limit(50)`). No load-more or pagination UI.

### Interaction Rules

1. The "Time" tab is never shown to managers or viewers — their session must not expose team member emails or actions via any UI path.
2. The team feed is loaded on tab mount (lazy, not at page load). Use a separate `useEffect` or conditional `useEffect` triggered when the active tab is "team" or when `isOwner` is confirmed.
3. Feed is read-only — no actions (dismiss, delete) on team entries.
4. The `ALERT_VIEWED` action must be added to `getActionLabel` in the existing map as `'alert.viewed': 'Visualizou alerta'`.
5. `ALERT_DISMISSED` maps to existing `'alert.dismissed': 'Alerta marcado como lido'` — no change needed.
6. `ALERT_DELETED` maps to existing `'alert.deleted': 'Alerta excluido'` — no change needed.
7. Entity type badge always shows "Alerta" for all team feed entries (all tracked actions are alert-scoped in this phase).
8. If the owner has no active members, the empty state card is shown inside the "Time" tab (see Copywriting Contract).

## States Reference

| State | Location | Trigger | Visual |
|-------|----------|---------|--------|
| Default — own activity | "Minha Atividade" tab | Page load | Existing activity log unchanged |
| Default — team feed | "Time" tab | Owner clicks tab | Card with feed entries, relative timestamps |
| Loading — team feed | "Time" tab | Tab mount before data arrives | "Carregando atividades do time..." centered in muted text |
| Empty — own activity | "Minha Atividade" tab | Zero audit logs for user | Clock icon + "Nenhuma atividade registrada" (existing) |
| Empty — team feed | "Time" tab | Zero alert-scope logs for any member | Clock icon + "Nenhuma atividade do time ainda" |
| No team tab | Activity page | Non-owner user | No TabsList rendered; "Minha Atividade" content shown directly |
| Error — team feed | "Time" tab | Supabase query failure | Inline error text: "Nao foi possivel carregar a atividade do time. Tente novamente." — no retry button |
