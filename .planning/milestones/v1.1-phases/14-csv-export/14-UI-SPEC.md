## Interaction Contracts

### Alerts Page — Date Filter Row

The alert list is the primary visual anchor of this page. The date filter row and export controls are secondary affordances positioned above the list.

Layout: `flex flex-wrap gap-2 justify-between items-center`

Left cluster (existing filter buttons, unchanged):
- "Todos (N)" — variant="default" when active, variant="outline" when inactive
- "Nao lidos (N)" — same pattern
- "Lidos (N)" — same pattern

Right cluster (new — added in this phase):
- Label "De" + `<Input type="date">` (w-36)
- Label "Ate" + `<Input type="date">` (w-36)
- Button "Aplicar Filtro" (variant="default", size="sm") — triggers date filter
- Button "Exportar CSV" with `<Download className="h-4 w-4 mr-2" />` (variant="outline") — triggers export of currently filtered alerts

Interaction rules:
1. Date inputs do not re-filter on keystroke — filter is applied only on "Aplicar Filtro" click
2. "Exportar CSV" exports exactly the alerts currently visible after all filters are applied (read/unread AND date range combined via AND)
3. "Exportar CSV" button is visible only when `alerts.length > 0` (matches existing guard)
4. Applying a date filter with no results in range shows the existing empty state card (no special empty state needed for date filter — reuse existing)
5. Clearing both date inputs and pressing "Aplicar Filtro" resets the date filter and shows all alerts (respecting the read/unread filter)

### Analytics Page — Export Button

The chart and metrics area is the primary visual anchor of this page. The export button is a secondary affordance positioned in the page header alongside the time-range tabs.

Placement: In the page header row, to the right of the `TabsList` containing 7d/30d/90d tabs

Layout: `flex items-center justify-between` wrapping the `TabsList` and the export button

Button spec: `<Button variant="outline" onClick={handleExport}><Download className="h-4 w-4 mr-2" />Exportar CSV</Button>`

Interaction rules:
1. Button exports `rawAlerts` state (alerts loaded for the current timeRange)
2. Button is always visible when analytics data is loaded (no guard needed since page always has a timeRange selected)
3. If `rawAlerts` is empty, `exportToCSV` already shows a native browser alert ("Nenhum dado para exportar") — no additional UI handling required
